模式匹配

朴素

❗❗时间复杂度

  • 主串长度为n, 模式串长度为m。
  • 最坏时间复杂度为:每个子串都要对比m个字符，共n−m+1n-m+1n−m+1个子串，复杂度为O((n-m+1)m) = O(n*m)

KMP

Partial Match （匹配表）

  • PM表算的是字符串的前缀和后缀的最长相等前后缀长度
编号 1 2 3 4 5
S a b c a c
PM 0 0 0 1 0 
         *          ❗❗**如果已匹配相等序列中存在最大相等前后缀(即首尾重合)，那么将子串向右滑动到和该相等前后缀对齐**(<u>理解关键</u>)
         *          当模式串与主串不匹配时，只需移动模式串即可
         *          右移位数 = 已匹配的字符数 - 对应的部分匹配值
         *          move = (j - 1) - PM[j - 1]

对PM改进（next[-1]）

    *  使用部分匹配值时，每当匹配失败，就去找它前一个元素的部分匹配值，这样使用起来不方便，所以将PM表右移一位，这样哪个元素匹配失败，直接看它自己的部分匹配值
编号 1 2 3 4 5
S a b c a c
next -1 0 0 0 1
  • 把PM改成了next
  • 右移以后第一个元素空缺，用-1填充。因为若是第一个元素匹配失败，则需将子串(模式串)右移一位**(move = 0 - (-1) = 1)**
  • 最后一个元素在右移过程中溢出，因为最后一个元素的匹配值是其下一个元素使用的，但显然已没有下一个元素，故舍去

最终改进(next[0])

  • 由上表可知 Move = (j - 1) - next[j], 相当于子串的比较指针j回退到 j = j - move = j - ((j - 1) - next[j]) = next[j] + 1
  • 因此为了计算方便，将上表next数组整体+1
编号 1 2 3 4 5
S a b c a c
next 0 1 1 1 2
  • 最终得到指针变化公式 j = next[j]
  • next[j] 的含义：在子串的第j个字符与主串发生失配时，则跳到子串的next[j]位置重新与主串当前位置进行比较

next[j]:{0,j=1maxk∣1<k<j且′p1...pk′=′pj−k+1pj−1′,当集合不空时1,其他情况next[j] :\begin{cases} 0, j = 1\\ max {k| 1 < k < j且'p_1...p_k'='p_{j-k+1}p_{j-1}'},当集合不空时\\ 1, 其他情况\end{cases}next[j]:⎩⎨⎧​0,j=1maxk∣1<k<j且′p1​...pk′​=′pj−k+1​pj−1′​,当集合不空时1,其他情况​

计算next[] (机算)

此时next[j]值为t,据此可以分两种情况来推导next[j+1]的值:

(1)若PjP_jPj​ = PtP_tPt​,则next[j+1] = t+1= next[i]+1;

(2)若PjP_jPj​ ≠PtP_tPt​,则循环的将t赋值为next[t]，直到t =0或者满足(1)为止，当t =0时, next[j+1]= 1。

  • 对(2)进行补充，当PjP_jPj​ ≠PtP_tPt​，但’P1P2...Pt−1P_1P_2...P_{t-1}P1​P2​...Pt−1​’ 和’Pj−t+1...Pj−1P_{j-t+1}...P_{j-1}Pj−t+1​...Pj−1​’

    这就等价于用下串去匹配上串，即下串P[t]的下标t = next[t];

KMP代码

❗当 j = 0时，i++表示主串右移， 然后j++表示j=1,重新指向模式串的第一个字符❗

❗❗❗时间复杂度

​ 主串长度为n, 模式串长度为m;

  • 求next数组的时间复杂度为O(m)
  • 模式匹配过程中最坏时间复杂度O(n), 匹配到尽头
  • 最坏时间复杂度为O(m+n)
