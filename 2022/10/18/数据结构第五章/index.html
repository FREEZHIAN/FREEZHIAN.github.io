第五章 树与二叉树

树的基本概念

  • 树是n个结点的有限集合， n = 0时，称为空树

    • 当 n > 1时，其余结点可分为m (m > 0)个互不相交的有限集合 T1,T2,...TmT_1,T_2,...T_mT1​,T2​,...Tm​,其中每个集合本身又是一棵树，并且称为根节点的子树

  • 结点

    • 根节点: 树只有一个根节点
    • 除了根节点外，任何一个结点都有且仅有一个前驱
    • 叶子节点: 没有后继的节点
    • 分支节点: 有后继节点

  • 结点的度：结点拥有的子树的数量

    • 度为0：叶子结点或者终端结点
    • 度不为0：分支结点或者非终端结点
      • 分支结点除去根结点也称为内部结点

  • 结点关系

    •       祖先结点
         *           根结点到该结点的唯一路径的任意结点

    •       子孙结点
         *              子孙节点是以某节点为根的子树中任一节点

    •       双亲结点
      * 根结点到该结点的唯一路径上最接近该结点的结点

    •       孩子结点
       			*   一个结点的直接后继结点称为该结点的孩子结点

    •       兄弟结点
      * 有相同双亲结点的结点

    •       层次，高度，深度，树的高度
      * 层次：根为第一层，它的孩子为第二层，以此类推
      * 结点的深度：根结点开始自顶向下累加
      * 结点的高度：叶节点开始自底向上累加
      * 树的高度（深度）：树中结点的最大层数

    •       有序树和无序树
         * 树中结点的各子树从左到右是有次序的，不能互换。否则称为无序树

树的性质

  • 1.树中的结点数等于所有结点的度数加1。

    • 证明：除根结点以外，每个结点有且仅有一个指向它的前驱结点。也就是说每个结点和指向它的分支一一对应。 假设树中一共有b个分支，那么除了根节点，整棵树就包含b个结点，所以整个树的结点数就是这b个结点加上根节点，设为n,则 n = b + 1。而分支数b也就是所有结点的度数。

  • 2.度为m的树、m叉树的区别

    度为m的树 m叉树
    任意结点的度<= m(最多m个孩子) 任意结点的度 <= m(最多m个孩子)
    至少有一个结点度 = m (有m个孩子) 允许所有的结点的度都< m
    一定是非空树，至少有m + 1 个结点 可以是空树

  • 3.度为m的树第i层至多有mi−1m^{i-1}mi−1个结点(i >=1)

    • 证明:(数学归纳法)

      首先考虑i = 1的情况:第一层只有根节点，即一个结点，带入式中成立； i = 2时, 最多有m个分支，也就是m个结点成立。

      …

      又因为树的度为m,所以对于第i-1层的每个结点，最多有m个孩子结点。所以第 i 层最多有 m∗mi−2=mi−1m*m^{i-2} = m^{i-1}m∗mi−2=mi−1个结点

  • 4.高度为h的m叉树至多有mh−1/m−1m^h-1/m - 1mh−1/m−1个结点

    • 证明: 等比数列求和公式: a+aq+aq2+...+aqn−1=a∗(1−qn)/(1−q)a + aq + aq^2 +...+ aq^{n-1} = a*(1-q^n)/(1-q)a+aq+aq2+...+aqn−1=a∗(1−qn)/(1−q)

      最多结点，那就要求每一层的每一个结点都有m个子结点 。即一共有m0+m1+m2+...+mh−1=(1−mh)/(1−m)=(mh−1)/(m−1)m^0+m^1+m^2+...+m^{h-1} = (1-m^h)/(1-m) = (m^h - 1)/(m - 1)m0+m1+m2+...+mh−1=(1−mh)/(1−m)=(mh−1)/(m−1)个结点

  • 5.高度为h的m叉树至少有h个结点。

    *      证明:因为m叉树可以为空，而且分支树可以小于等于m。又因为其存在高度h,说明不为空，故可以至少可以有h个结点

  • 5.高度为h、度为m的树至少有h+m-1个结点。

    • 证明:因为是度为m的树，故至少存在一个度为m的结点，也就是该结点有m个分支。然后可以让第h - 1层满足该条件，剩下的 h-2层只有一个结点。因此一共有 h - 2 + m + 1 = h + m - 1个结点

  • 6.具有n个结点的m叉树的最小高度为⌈logm(n(m−1)+1)⌉\lceil logm(n(m - 1)+ 1)\rceil⌈logm(n(m−1)+1)⌉

    • 证明: 为让高度最小，需让每一层的结点尽可能的多，也就是所有结点都有m个孩子

      前h-1层共有mh−1−1/(m−1)m^{h-1}-1/(m-1)mh−1−1/(m−1)个结点

      前h层共有mh−1/(m−1)m^h-1/(m-1)mh−1/(m−1)个结点

      $ m^{h-1}-1/(m-1) < n <= m^h-1/(m-1)$ ->

      $ m^{h-1} < n*(m-1) + 1 <= m^h$ ->

      h−1<logmn∗(m−1)+1<=hh-1 < log_m^{n*(m-1)+1} <= hh−1<logmn∗(m−1)+1​<=h ->

      故hmin=⌈logm(n(m−1)+1)⌉h_{min} = \lceil logm(n(m - 1)+ 1)\rceilhmin​=⌈logm(n(m−1)+1)⌉

二叉树的基本概念

  • 定义

    • 二叉树是n（n≥0）个结点的有限集合：
      ① 或者为空二叉树，即n=0。
      ② 或者由一个根结点和两个互不相交的被称为根的左子树
      和右子树组成。左子树和右子树又分别是一棵二叉树。
      • 1.每个结点最多有两棵子树。
      • 2.左右子树有顺序

  • 二叉树的五种基本形态：

    • 1.空树
    • 2.只有一个根结点
    • 3.根结点只有左子树
    • 4.根结点只有右子树
    • 5.根结点既有左子树又有右子树

  • 二叉树VS度为2的有序树
    * 度为2的有序树至少与3个结点，而二叉树可以为空
    * 度为2的有序树的孩子的左右次序是相对于另一个孩子而言的，若某个结点只有一个孩子，则这个孩子就无须区分其左右次序，而二叉树无论其孩子数是否为2，均需确定其左右次序，即二叉树的结点次序不是相对另一结点而言，而是确定的

二叉树的存储结构

  • 顺序存储

    • 二叉树的顺序存储结构就是用一组地址连续的存储单元依次自上而下、自左至右存储完全二叉树上的结点元素。

  • 链式存储

    • 二叉树每个结点最多两个孩子，所以设计二叉树的结点结构时考虑两个指针指向该结点的两个孩子。

    • n个结点的二叉链表共有n+1个空链域

      证明: 由 结点总数 = 总度数 + 1 -> 结点总数 = 总分支数 + 1

      故一共会有 n-1个非空指针， 每个节点一共有2个指针域，n个结点共有 2n 个指针域。 空链域 = 2n- (n-1) = n + 1

二叉树的遍历

层次建树

  • 层次建树的思想是运用一个辅助队列(没有头结点)，把每一个输入的值放入队列中，如果队头元素(pcur)的左右结点都由元素连接，那么就出队，对下一个元素进行操作。边访问队列，边建树
  • 队列里放的是树地址，这样更节省空间
  • pcur指向当前为根(子树)的结点，若无左子树，就指向刚输入的元素；若无右子树，就指向刚输入的元素；若已满，则指向队列的下一个结点，重复上述步骤

层次遍历

  • 层次遍历算法思想:
    • ①初始化一个辅助队列
    • ②根结点入队
    • ③若队列非空，则队头结点出队，访问该结点，并将其左、右孩子插入队尾（如果有的话)
    • ④重复③直至队列为空

先序遍历(根左右)

  • 先序遍历递归
  • 先序遍历非递归
  • 利用一个栈来存放结点，右孩子先入栈，左孩子后入栈
  • 和王道书差不多的版本
  • 🐱‍👓先序遍历的思想
    • 1.首先先序遍历是按照根左右的顺序进行遍历的，因此先访问根，但是根必须得入栈，否则就不能根据跟结点访问右子树了。
    • 2.访问完根节点之后，在访问左节点，就是一路向左
    • 3.当左节点为空时，就到了访问右节点的时刻了
    • 4.取出在栈顶的根节点，通过根节点访问右节点
    • 重复以上操作

中序遍历(左根右)

  • 中序遍历递归
  • 中序遍历非递归
  • 🐱‍👤中序遍历的思想
    • 1.首先中序遍历是按照左根右的顺序进行遍历的，因此先访问左子树，不断压栈，一路向左
    • 2.当左子树为空时，取出栈顶(也就是根节点)，进行输出
    • 3.访问右子树
    • 4.重复上述操作

后序遍历(左右根)

  • 后序遍历递归
  • 后序遍历非递归
  • 🐱‍👤后序遍历的思想
    • 1.首先后序遍历是按照左右根的顺序进行遍历的，因此先访问左子树，不断压栈，一路向左
    • 2.当左子树为空时，取出栈顶(也就是根节点)，判断其是否存在右子树
    • 3.若存在右子树，且未被访问过，便可以访问右子树，对其进行标记，重复1，2，3操作
    • 4.若存在右子树，但被访问过了，便可访问根节点，出栈
    • 5.若不存在右子树，便可访问根节点，出栈

由遍历序列构造二叉树

  • 结论: 前序、后序、层序序列的两两组合无法唯一确定一棵二叉树

线索二叉树(+遍历)

  • N个结点的二叉链表，每个结点都有指向左右孩子的
    结点指针，所以一共有2N个指针，而N个结点的二叉
    树一共有N-1条分支，也就是说存在2N-(N-1)=N+1个空指针。
  • 大量的空余指针能否利用起来？
    • 指向前驱和后继的指针称为线索，加上线索的二叉链表就称为线索链表，相应的二叉树就称为线索二叉树
    • 对二叉树以某种次序遍历使其变为线索二叉树的过程就叫做线索化

前序线索化

  • 防止陷入死循环 if(p->ltag == 0) preThread(p->lchild, pre);
    • 先序遍历是按照根左右的顺序进行遍历的，先对根进行线索化，倘如根无左子树，因此线索化，指向前驱。如果不加上if()语句，那么程序就会对左子树线索化，但左子树已被线索化，指向根，因此会陷入死循环

前序线索

找前驱

  • 先序遍历中，左右子树中的结点只可能是根的后继,不可能是前驱(因为先遍历根)

    • 2.若p->ltag == 0,则其必有左孩子，因为先序是按照根左右进行线索化的，所以p的前驱一定在其父节点找
    • 4.如果p为根节点，则没有前驱
    • 4.如果能找到其父节点

找后继

  • 在先序线索二叉树中找到指定结点*p的先序后继next

    • 1.若p->rtag == 1,则next = p->rchild

    • 2.若p->rtag == 0必比有右孩子
      • 1.若p有左孩子，则先序后继为左孩子
      • 2.若p无左孩子，有右孩子，则先序后继为右孩子

找前驱

  • 在中序线索二叉树中找到指定结点*p的中序前驱pre

    • 1.若p->ltag == 1,则pre = p->lchild
    • 2.若p->ltag == 0 必有左孩子

    • 3.按照左根右遍历
      • pre = p的左子树中最右下的结点
        • 原因：已知p必有左孩子，若以左孩子为根的子树有左孩子，无右孩子，那么p的前驱为左孩子结点；若以左孩子为根的子树有左孩子（无左孩子），有右孩子，则p的前驱为p的左子树最右下的结点

找后继

  • 在中序线索二叉树中找到指定结点*p的中序后继next
    • 1.若p->rtag == 1,则next = p->rchild
    • 2.若p->rtag == 0,则必有右孩子
      • next = p的右子树中最左下的结点
        • 原因：已知p必有右孩子，若以右孩子为根的子树无左孩子，有右孩子，那么p的后继为右孩子结点；若以右孩子为根的子树有左孩子，（有无）右孩子，则p的后继为p的右子树最左下的结点

树、森林的遍历

树的遍历

  • 先根遍历(若树非空，先访问根结点，在依次对每棵子树进行先根遍历)

    • 树的先根遍历序列与这棵树相应二叉树的先序序列相同
    • 树的先根遍历相当于深度优先遍历

  • 后根遍历(若树非空，先依次对每棵子树进行后根遍历，最后在访问根结点)

    • 树的后根遍历序列与这颗树相应二叉树的中序序列相同

  • 层序遍历

    • ①若树非空，则根节点入队
    • ②若队列非空，队头元素出队并访问，同时将该元素的孩子依次入队
    • ③重复②直到队列为空
    • 相当于广度优先遍历

森林的遍历

  • 先序遍历(等同于对各个树进行先根遍历)

    • 若森林为非空，则按如下规则进行遍历:
    • 访问森林中第一棵树的根结点。
    • 先序遍历第一棵树中根结点的子树森林。
    • 先序遍历除去第一棵树之后剩余的树构成的森林。
    • 相当于对二叉树进行先序遍历

  • 中序遍历

    • 若森林为非空，则按如下规则进行遍历:
    • 中序遍历森林中第一棵树的根结点的子树森林。
    • 访问第一棵树的根结点。
    • 中序遍历除去第一棵树之后剩余的树构成的森林。
    • 相当于对各个树进行后根遍历
    • 相当于对二叉树中序遍历

哈夫曼树

  • 在含有n个带权叶结点的二叉树中，其中带权路径长度(WPL)最小的二叉树称为哈夫曼树，也称最优二叉树

  • 前缀码
    • 若没有一个编码是另一个编码的前缀，则称这样的编码为前缀编码
  • 哈夫曼树的性质
    • 每个初始结点最终都成为叶结点，且权值越小的结点到根结点的路径长度越大
    • 哈夫曼树的结点总数为2n - 1
    • 哈夫曼树中不存在度为1的结点。
    • 哈夫曼树并不唯一，但WPL必然相同且为最优
