增（插入）

❗ 易错点:

 * i 表示的是位置，可以在 **1- length + 1** 的任何位置插入元素 
 * 顺序表中的元素的下标是 **0 - length - 1**
 * 因为是插入，所以必然会增加一个元素，所以 `j = L.length` 指向了表尾后一个元素，作为缓冲
 * 只要不是在表尾增加元素 `j >= i` 恒成立

​ 时间复杂度：

  • 最好情况: 在第一个位置增加元素O(1)

  • 最坏情况: 在最后一个位置增加元素O(n)

  • 平均情况: 一共 n + 1 个位置， 假设插入概率都相同，$$p = 1 / (n + 1)$$ 第一个位置移动 n 次，第二个位置移动 n - 1次… 第n+1位置移动0次,平均循环次数 = $$np + (n - 1)p + …+1*p = n * (n + 1) /2 * 1/(n + 1) = n / 2$$

    因此平均复杂度为 O(n)

删（删除）

​

❗易错点:

  • 删除元素位置的合法范围是1 - length

时间复杂度:

  • 最好情况: 删除最后一个元素O(1)

  • 最坏情况: 删除第一个元素，需移动 n - 1次 O(n)

  • 平均情况: 一共 n 个位置， 假设删除概率都相同，$$p = 1/n$$ 第一个位置移动 n - 1 次，第二个位置移动 n - 2次… 第n位置移动0次,平均循环次数 = $$(n - 1)p + …+1*p = n * (n - 1) /2 * 1/n = (n - 1)/ 2$$

    因此平均复杂度为 O(n)

查(查找)

  • ✨ 按位查找

​ 时间复杂度: O(1)

  • ✨ 按值查找

❗易错点:

  • 当按值查找的元素的类型为结构体类型时，不可以直接比较，需要按照分量一一比较

​ 时间复杂度:

  • 最好情况: 在第一个位置找到元素O(1)

  • 最坏情况: 在最后一个位置找到元素O(n)

  • 平均情况: 一共 n 个位置， 假设插入概率都相同，$$p = 1/n$$ 第一个位置循环1次，第二个位置循环2次，第n个位置循环n 次,平均循环次数 = $$1*p + …+ (n - 1)p + np = n * (n + 1) /2 * 1/(n + 1) = n / 2$$

    因此平均复杂度为 O(n)

总结

🐱‍👓 逻辑结构

  • 顺序表和链表都属于线性表，都是线性结构

🐱‍🚀 存储结构

  • 顺序表是顺序存储
  • 顺序表的优点:支持随机存取，存储密度高
  • 顺序表的缺点:大片连续空间分配不方便，改变容量不方便
  • 链表是链式存储
  • 链表的优点:离散的小空间分配方便，改变容量方便
  • 链表的缺点:不可随机存取，存储密度低

🐱‍👤 基本操作

创

  • 顺序表分为静态分配与动态分配
  • 顺序表需要预分配大片连续空间。若分配空间过小，则之后不方便拓展容量；若分配空间过大，则浪费内存资源
  • 静态分配:用的是(容量不可改变)的静态数组
  • 动态分配:用的是(容量可改变，但需要移动大量元素，时间代价高)的动态数组
  • 链表只需要分配一个头节点(也可以不要头节点，只声明一个头指针)，之后方便拓展

销

  • 顺序表-静态分配:静态数组只需修改length = 0即可。系统会自动回收空间
  • 顺序表-动态分配:动态数组(malloc,free)需要手动free (malloc 和 free 成对出现)
  • 链表:依次删除各个节点(free)

增 + 删

  • 顺序表插入/删除元素要将后续元素都 后移/前移
  • 顺序表的增删操作时间复杂度为O(n),时间开销主要来自移动元素
  • 顺序表若数据元素很大，则移动的时间代价很高
  • 链表插入/删除元素只需修改指针即可
  • 链表的时间复杂度为O(n),时间开销主要来自查找目标元素
  • 链表查找元素的时间代价更低

查

  • 顺序表:按位查找-O(1)
  • 顺序表:按值查找-O(n)，若元素有序，利用二分可达到$$ O(log2^n)$$
  • 链表:按位查找-O(n)
  • 链表:按值查找-O(n)

🎉表长难以预估、经常要增加/删除元素 ——链表

🎉表长可预估、查询（搜索）操作较多 ——顺序表
